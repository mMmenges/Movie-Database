Documentation for myFlix

This web application gives users access to information about different movies, directors, and genres.
Users can sign up, update their personal information, and create a list of their favorite movies.

HTTP request details

1. General App features

  • Method type: GET
  • Query Perameters: '/'
  • Data in Body of Request: None
  • Response: Welome message for app

2. Returning a list of ALL movies to the title'
  • Data in Body of Request: None
  • Response: A JSON object holding the movie detials for the specific title. E.g.
    {title: "film name", description: "description", director: {name: "director name", bio: "director bio", birth: "birth year"}, genre: {Name: "genre", description: "genre description"}, release: "year", Staring: [actors], imageURL: "URL", Featured: "Yes or No"}

    • 4. Returning data about a genre (description) by name/title (e.g., “Thriller”)

    • Method type: GET
    • Query Perameters: '/movies/Genres/Name'
    • Data in Body of Request: None
    • Response: All JSON objects relevant to specific genre selected in format e.g.
      {Name: "Genre Name", Description: "genre description"}

    5. Returning data about a director (bio, birth year) by name

    • Method type: GET
    • Query Perameters: '/Director/Name'
    • Data in Body of Request: None
    • Response: All JSON data relevant to specific director (including DOB, and other details if listed) selected in format e.g.
      {name: "director name", bio: "director bio", birthyear: "yyyy" }

    6. Allowing new users to register

    • Method type: POST
    • Query Perameters: '/users'
    • Data in Body of Request: A JSON object holding the necessary data to create a new profile e.g.
      {name: "test name", email: "test@test.com", password: "test password", dateOfBirth: "test date"}
    • Response: A JSON object holding the name and email data for the new profile (not password or DOB). Now object will now include and display "id" as well e.g.
      {id: "new id", name: "test name", email: "test@test.com", password: "test password", dateOfBirth: "test date"}

    7. Creating a webpage for each specific user

    • Method type: GET
    • Query Perameters: '/users/:Username'
    • Data in Body of Request: None
    • Response: Welcome text message using user.name

    8. Allowing users to update their user info (username, password, e-mail, date of birth)

    • Method type: PUT
    • Query Perameters: '/users/:Username'
    • Data in Body of Request: A JSON object holding the necessary data being updated in the profile e.g.
      {name: "new test name", email: "newtest@test.com", password: "newtest password", dateOfBirth: "new test date"}
    • Response: A text message specifying that the update was successful for specific field.

    9. Allowing users to add a movie to their favorites list

    • Method type: POST
    • Query Perameters: '/users/username/favorites/_id'
    • Data in Body of Request: None (movie object already exists in app)
    • Response: A text message confirming movie successfully added to list

    10. Allowing users to remove a movie from their list of favorites

    • Method type: DELETE
    • Query Perameters: '/users/username/favorites/_id'
    • Data in Body of Request: None
    • Response: A text message confirming movie successfully deleted from list.

    11. Allowing existing users to deregister

    • Method type: DELETE
    • Query Perameters: '/users/:username'
    • Data in Body of Request: None
    • Response: A text message confirming successful deletion of account.
